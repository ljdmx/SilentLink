# SilentLink | 零存储端到端加密通信系统

SilentLink 是一款专为极致隐私设计的点对点（P2P）通信工具。基于 WebRTC 技术栈，通过浏览器原生硬件加固加密，确保您的音视频通话和文件传输仅存在于参与者的内存中，真正实现“阅后即焚，无痕交流”。

---

## 🌟 核心特性

- **🔒 绝对隐私 (E2EE)**：采用 AES-256-GCM 算法对所有流量进行端到端加密，密钥由参与者协商产生，不经过任何中转服务器。
- **🚀 零存储架构**：全应用运行于浏览器内存，关闭网页即物理销毁所有密钥与聊天记录。
- **🛡️ 隐私防护滤镜**：内置 WebGL 实时滤镜（模糊、马赛克、纯黑），在音视频发出前即完成本地脱敏。
- **📂 安全文件传输**：支持大文件分片加密传输，同样遵循点对点协议，绕过任何中心化存储。
- **📱 响应式设计**：完美适配手机与 PC 屏幕，移动端优化聊天与控制栏交互体验。
- **👻 自动保护**：检测到浏览器标签失焦或最小化时，自动遮盖视频画面。

## 🚀 快速上手

### 1. 部署教程
本项目为纯前端应用，无需后端数据库或 API 支持。
- **静态托管**：将所有文件上传至 Nginx, GitHub Pages, Vercel 或 Netlify。
- **环境要求**：必须在 **HTTPS** 环境下运行（WebRTC 强制要求安全上下文以调用媒体硬件）。

### 2. 使用指南
1. **初始化**：点击“初始化安全频道”，设置名称、房间 ID 和**会话口令**（双方必须一致）。
2. **建立隧道（手动握手）**：
   - **发起方**：生成并发送加密请求链接给对方。
   - **接收方**：打开链接或注入请求代码，生成并回传“响应代码”。
   - **开启**：发起方填入响应代码，点击“开启隧道”激活 P2P 链路。
3. **物理退出**：点击“物理退出”将立即销毁本地媒体轨道、清空加密密钥并重定向回首页。

## 🛠️ 技术架构与实现细节

### 1. 密钥派生与加密架构 (Web Crypto API)
系统安全性建立在浏览器原生硬件加固的 **Web Crypto API** 之上。
- **密钥派生**：使用 `PBKDF2` 算法，配合 100,000 次哈希迭代及 `RoomID` 作为盐值，将用户口令转化为 **AES-256-GCM** 对称密钥。
- **分片加密**：文件传输采用 16KB 分片策略，每个 Chunk 独立加密并封装 IV 头部，通过 WebRTC DataChannel 的有序模式发送。

### 2. 无服务器信令交换 (Manual Signaling)
为了达成“零服务器存储”的目标，SilentLink 放弃了传统的 WebSocket 信令服务器。
- **手动握手**：通过 Base64 编码的 SDP 载荷实现信令同步，杜绝了信令服务器收集用户元数据（IP、在线时间、联系链）的可能性。
- **NAT 穿透**：握手完成后，双方通过公开的 STUN 服务器发现公网路径，直接建立双向 UDP 隧道。

### 3. 隐私保护滤镜 (WebGL/Canvas Pipeline)
SilentLink 实现了“预发脱敏”机制，画面在离开本地前已完成数学处理。
- **渲染管线**：本地流被捕获到隐藏 `<canvas>`，利用 `requestAnimationFrame` 在帧级执行马赛克或模糊算法。
- **物理切断**：当开启“纯黑”模式时，Canvas 直接填充黑色像素，确保网络上传输的数据已经过物理层面的脱敏。

### 4. 动态资源销毁机制 (Zero Persistence)
系统严格遵循“阅后即焚”的内存管理逻辑：
- **同步退出**：一方退出时发送 `session-terminate` 特权信令。
- **资源回收**：调用 `RTCPeerConnection.close()`，执行 `MediaStreamTrack.stop()` 关闭硬件供电，并清除 `CryptoKey` 引用，释放所有内存 Blob。

## 🛡️ 安全防护特性

- **窗口失焦保护 (Focus Protection)**：监听 `visibilitychange` 事件。当用户切换标签页或最小化时，系统立即进入隐私屏蔽模式，暂停 Canvas 绘制并覆盖毛玻璃遮罩。
- **抗截屏干扰 (Visual Deception)**：UI 层使用高饱和度动态噪声背景和特殊图层叠加，旨在干扰 OCR 文字识别。
- **硬件加速优化**：音视频编码由浏览器底层硬件加速器处理，即便执行实时 WebGL 滤镜也能保持低延迟。
